# Известные проблемы и ограничения

Зафиксировано **до рефакторинга**. Исправление — по [`refactoring_plan.md`](../refactoring_plan.md), не «случайно» при других задачах.

## Баги

### 1. Даты через границу года — исправлено (2026-06)

**Файл:** `author_today/domain/models.py` → `parse_dd_mm_columns`, `ReadSnapshot.from_stats_table`

Год увеличивается при «откате» месяца в заголовках `DD.MM` (от `period_start.year`).

---

### 2. CSV/JSON только за последний месяц при длинном периоде — документировано (2026-07)

**Файл:** `author_today/pipeline/sync_reads.py` → `sync_reads_by_period`

При `needs_monthly_chunks()` в `-o` / `--json` попадает таблица **последнего** chunk; в MS SQL и в `data/raw` (автосохранение без явного `--json`) сохраняются **все** chunks.

**Исправление (interim):** при `-o` / `--json` на период > 1 месяца CLI печатает предупреждение в stderr. Полный merge в один файл — см. ADR-005 (вариант A), не реализован.

**Воспроизведение:** период > 1 календарного месяца + `-o out.csv` → предупреждение + CSV только за последний месяц.

---

### 3. Расхождение JSON vs MSSQL в отчётах

**Файлы:** `analyze/funnel.py`, `analyze/funnel_compare.py`

- Из JSON: `chapter_order` = индекс в массиве / порядок первого появления
- Из MSSQL: `chapter_order` из `chapter_reads` (как при сохранении с сайта)

При одном файле JSON и БД результаты воронки должны совпадать только если порядок глав стабилен.

---

### 4. Kendo Grid: не все дни месяца в одном снимке — исправлено (2026-06)

**Файл:** `author_today/parse/kendo_grid.py`

Раньше парсер читал только **видимые** колонки таблицы без горизонтальной прокрутки → при загрузке месяца в БД могли попасть только последние ~10 дней (например, с 21.06).

**Исправление:** прокрутка `.k-grid-content`, сбор видимых колонок по `getBoundingClientRect`, сопоставление строк без `zip` (строка «Часть» только в locked-колонке). Перезагрузите период; неполный run удалите через `delete_runs.py`.

**Симптом смещения глав (run 80, 2026-06):** у «Главы 1» в БД оказывались числа «Главы 2» — из-за `zip(locked_rows, scroll_rows)` со строкой «Часть».

**Симптом «только 15 глав»:** Kendo виртуализирует строки по вертикали (~15 в DOM). Исправление: чтение `dataSource` через jQuery (если доступно) и вертикальная прокрутка `.k-grid-content` с горизонтальным сбором дат.

## Ограничения модели (не баги, но важно для интерпретации)

### Просмотры ≠ уникальные читатели

Один пользователь может дать просмотры нескольким главам → `% от пред.` может быть **> 100%**.

### Дубли fetch_runs в БД

Повторная загрузка того же периода **добавляет** новый run; аналитика **суммирует** views. Очистка: `scripts/delete_runs.py` по `work_id` + диапазон `fetched_at`.

### p-value при малом числе дней

Меньше 2 дней с ненулевой базой в периоде → p-value = `—`.

### Нулевая дисперсия

Если дневные % константны в обоих периодах, p-value = 1 (одинаково) или 0 (разные средние).
`scipy.ttest_ind` в этом случае даёт NaN — `hypothesis_tests.welch_ttest_pvalue` уходит в fallback.

### scipy не в requirements.txt

`hypothesis_tests.py` работает без scipy (fallback); при установленном scipy используется `ttest_ind`.
