# Решения на время рефакторинга (ADR)

Краткие архитектурные решения. Нумерация сохраняется при добавлении новых.

---

## ADR-001: Имя идентификатора книги в Python

**Статус:** принято, **реализовано** (2026-06)
**Контекст:** `work_id`, `book_id`, `AT_WORK_ID` смешаны в CLI и коде.  
**Решение:** в Python, CLI и документации — **`book_id`**. Колонка БД `fetch_runs.work_id` без переименования; маппинг в `mssql_repo` только.  
**CLI:** `--book-id` основной; `--work-id` — deprecated alias минимум на один релиз.  
**Последствия:** правки argparse, README, без миграции SQL.

## ADR-002: Каноническая модель для аналитики

**Статус:** принято, реализовано (2026-06, §3)  
**Контекст:** JSON парсился по-разному в funnel и funnel_compare.  
**Решение:** единая точка — **`ReadSnapshot`**; загрузка `from_json`, `from_stats_table`, `mssql_repo.load_snapshot()`. Отчёты — `funnel_from_snapshot()` / `daily_matrix()`.  
**Последствия:** legacy JSON-парсеры — тонкие обёртки над snapshot.

## ADR-003: SQL только в storage

**Статус:** принято, реализовано (2026-06, §2)  
**Контекст:** SQL в analyze и scripts.  
**Решение:** запросы чтения/удаления — методы **`MssqlReadRepository`** (или `mssql/queries.py`). Analyze не импортирует `pyodbc` напрямую.  
**Последствия:** проще менять схему; mock-репозиторий для тестов.

## ADR-005: Поведение multi-month export

**Статус:** принято (вариант C, 2026-07)

| Вариант | Описание |
|---------|----------|
| A | `merge_stats_tables()` — один CSV/JSON на весь период |
| B | Запрет `-o`/`--json` при >1 month с понятным сообщением |

**Контекст:** нужен визуальный интерфейс для отчётов; рассматривались PyQt, Django.

**Последствия:** зависимость `requirements-ui.txt`; UI после рефакторинга §2–§3; см. [`ui_streamlit.md`](ui_streamlit.md).

## ADR-012: MS SQL — источник правды, JSON — legacy

**Статус:** принято (2026-06)  
**Контекст:** данные накапливаются в MS SQL; файлы `data/raw/reads_*.json` остались от промежуточной версии пайплайна, когда БД была опциональной.  
**Решение:**

- **Отчёты, UI, сравнение периодов** — только из MS SQL
- JSON **не удалять** сразу: код парсинга остаётся для тестов и отладки
- **Скрыть** из пользовательского пути: нет выбора JSON в Streamlit; CLI `--json` только при `AT_ENABLE_LEGACY_JSON=yes`
- Загрузка с сайта по-прежнему может писать raw JSON как побочный артефакт — до отдельной задачи на выпиливание

**Последствия:** `Settings.enable_legacy_json`; рефакторинг §3 — **MSSQL → ReadSnapshot**, а не продакшен-путь через JSON; см. `refactoring_plan.md`.
