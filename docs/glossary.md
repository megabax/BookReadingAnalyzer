# Глоссарий

Единый словарь терминов для кода, документации и рефакторинга.

## Идентификаторы книги

| Термин | Где используется | Значение |
|--------|------------------|----------|
| **workId** | URL author.today, API сайта | ID произведения на сайте |
| **book_id** | Python (`ReadSnapshot`, `Settings`, отчёты) | То же, что workId; предпочтительное имя в коде |
| **work_id** | MS SQL (`fetch_runs.work_id`, `books.id`) | То же значение; имя колонки в БД |
| **AT_BOOK_ID** | `.env` | Переменная окружения → `Settings.book_id` (предпочтительно) |
| **AT_WORK_ID** | `.env` | Устаревший alias для `AT_BOOK_ID` |

**Правило после рефакторинга:** в Python и CLI — только `book_id`; `work_id` — только на границе SQL-репозитория.

## Источник данных

| Термин | Описание |
|--------|----------|
| **Источник правды** | **MS SQL** — все отчёты и UI читают из БД |
| **JSON-снимок** | Файл `data/raw/reads_*.json`; устаревший артефакт загрузки, скрыт по умолчанию |
| **AT_ENABLE_LEGACY_JSON** | `yes` — разрешить `--json` в CLI и JSON в `services/reports` (отладка) |
| **ReadSnapshot** | Доменная модель снимка; целевой путь: **БД → repo → ReadSnapshot → отчёты** (не из файла) |

## Главы и порядок

| Термин | Описание |
|--------|----------|
| **chapter_name** | Название строки в таблице сайта («Глава 1», «Интерлюдия 1», «Страница книги») |
| **chapter_order** | Порядок главы на сайте: 1…N сверху вниз в Kendo Grid |
| **step_num** | Порядковый номер шага в воронке (1…N после фильтров), не совпадает с `chapter_order` при `--skip-book-page` |
| **site_chapter_order** | `chapter_order` на сайте / в БД (поле в `FunnelStep`) |
| **baseline / base-order** | Глава, от которой считается 100% в колонке «% от базы» |

## Периоды и загрузки

| Термин | Описание |
|--------|----------|
| **period_start / period_end** | Календарный интервал запроса к сайту и фильтра отчётов |
| **read_date** | Конкретный день в матрице просмотров (`chapter_reads.read_date`) |
| **fetch_run** | Одна загрузка: запись в `fetch_runs` + все строки `chapter_reads` с `run_id` |
| **fetched_at** | Метка времени сохранения снимка в БД |
| **chunk** | Подпериод при помесячной загрузке (`split_period_into_months`) |

## Модели данных

| Термин | Класс / таблица |
|--------|-----------------|
| **StatsTable** | Сырой парс страницы: заголовки `DD.MM`, строки глав |
| **ReadSnapshot** | Нормализованный снимок: `dates[]`, `chapters[]`, `values[][]` |
| **DailyMatrix** | `date → {chapter_order → (name, views)}` — для сравнения воронок |
| **FunnelStep** | Один шаг воронки: просмотры, % от базы, % от пред., падение |

## Отчёты

| Отчёт | Скрипт | Суть |
|-------|--------|------|
| **Воронка** | `report_funnel.py` | Сумма просмотров за период по главам |
| **Сравнение воронок** | `report_funnel_compare.py` | Дневные % от базы, μ, σ, p-value (Welch) |
| **Удаление run** | `delete_runs.py` | Очистка дублей в БД по `work_id` + диапазон `fetched_at` |

## Статистика

| Термин | Описание |
|--------|----------|
| **μ (mean)** | Среднее дневных % за период по одной главе |
| **σ (sigma)** | Несмещённое стандартное отклонение дневных % |
| **p-value** | Welch t-test: значимость различия μ между периодами A и B |
| **Welch t-test** | t-тест для независимых выборок с разной дисперсией |

## Не путать

- **`hypothesis_tests.py`** — модуль расчёта p-value (Welch), **не** каталог unit-тестов (раньше `stats_test.py`).
- **views** — число просмотров с сайта, **не** уникальные читатели.
- **воронка дочитывания** — сравнение просмотров по порядку глав, **не** классическая conversion-воронка по пользователям.
